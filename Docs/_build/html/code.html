routes.users.createUser()¶

Create a new user and add to the database, use Bcrypt to hash password

Parameters:
  • email (str) – the email of the user
  • username (str) – (optional) the username of the user
  • password (str) – the password of the user
  • firstName (str) – the first name of the user
  • lastName (str) – (optional) the last name of the user
Returns:

“User Successfully Created”, status code
Return type:

str, int
Raises:
  • KeyError – if the input is not provided by the user
  • sqlalchemy.exc.IntegrityError – if the user already existed in the database

Chores¶

routes.chores.assignUserOrDeadlineToChore()¶

Assign a user and/or deadline to a chore.

Notes: For initial assignment, email and deadline are required parameters. For editing the deadline or assigned user later, email and deadline are optional parameters. Postcondition: both user and deadline must not be null.

Parameters:
  • id (int) – the unique ID corresponding to the target chore
  • email (str) – the email of the user who will be assigned to the chore
  • deadline (str) – the new deadline for the chore (format: “mm/dd/yyyy”)
Returns:

a message confirming that the user and deadline have been set, status code
Return type:

str, int
Raises:
  • KeyError – chore ID was not specified
  • sqlalchemy.orm.exc.NoResultFound – chore ID does not exist, or user email does not exist
